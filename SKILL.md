# 小红书图文生成

根据用户提供的 Markdown 内容，自动从图片库匹配大学图片，生成小红书风格的 HTML 图文卡片（3:4竖版，2K分辨率）。内容驱动，形式跟随内容。

## 基本原则

**语言一致性**：生成的内容文字应与用户输入的文字保持一致，**除非用户明确指定需要特定语言**。例如：用户输入中文则保持中文，用户输入英文则保持英文。

## 输入与输出

### 输入
- **Markdown 文件**：包含要转化为图文的内容

### 输出
- **HTML 文件**：每张卡片一个 HTML 文件，可直接在浏览器预览
- 用户可自行将 HTML 转换为 PNG（如使用截图工具、Puppeteer 等）

### 图片库（预先配置）

**大学图片库**：
```
references/universities/
├── 浙江大学/
│   ├── logo.png           # 学校 logo
│   ├── campus-1.jpg       # 校园图片
│   ├── campus-2.jpg
│   └── ...
├── 南京大学/
│   ├── logo.png
│   ├── campus-1.jpg
│   └── ...
└── [其他大学]/
```

**背景图片库**：
```
references/bg/
├── bg_1.jpg   # 简约纯色背景（浅色）
├── bg_2.jpg   # 简约纯色背景（深色）
├── bg_3.jpg   # 渐变背景
├── bg_4.jpg   # 纹理背景
├── ...
├── bg_37.png  # 艺术边框背景
├── bg_38.png  # 几何图案背景
└── ... (共50张)
```

**背景图片选择规则**：
- 青春活力风格 → 选择 bg_1~bg_10（明亮暖色系）
- 专业权威风格 → 选择 bg_11~bg_20（深蓝、金色系）
- 简约现代风格 → 选择 bg_21~bg_30（白色、浅灰、几何）
- 学院复古风格 → 选择 bg_31~bg_36（复古纹理、边框）
- 清新自然风格 → 选择 bg_37~bg_50（自然、艺术边框）

## 工作流程

### 1. 分析 Markdown 内容

**解析用户提供的内容**：
- 内容主题是什么？（大学介绍、宿舍条件、专业推荐、招生数据...）
- **涉及哪所大学？** → 从内容中识别大学名称
- 核心信息点有哪些？
- 内容的情感基调？（专业/活泼/温馨/严谨...）
- 信息密度如何？（要点式/叙述式/数据密集型...）
- 内容结构如何？（标题层级、列表、表格...）

**不要预设固定模块**，根据实际内容灵活规划。

### 2. 匹配图片资源

**自动匹配流程**：

1. 从 Markdown 内容中识别大学名称
2. 在 `references/universities/` 中查找对应文件夹
3. 读取该大学的图片资源：
   - `logo.png` - 用于封面品牌标识
   - `campus-*.jpg` - 根据内容主题选择匹配的校园图片

**图片选择原则**：
- 封面：使用标志性建筑或最美校园图片
- 内容图：根据内容主题匹配（宿舍内容配宿舍图、食堂内容配食堂图...）
- 如无法精确匹配，选择通用校园风景图

**如果图片库中没有对应大学**：
- 提示用户需要先添加该大学的图片
- 或使用通用占位图/纯色背景

### 3. 确定视觉方案

**根据内容选择风格**（智能推荐或用户确认）：

| 内容特征 | 推荐风格 | 配色倾向 |
|----------|----------|----------|
| 年轻活力、校园生活 | 青春活力 | 暖橙、明亮 |
| 数据、排名、官方介绍 | 专业权威 | 深蓝、金色 |
| 简洁信息、现代感 | 简约现代 | 白色、紫蓝 |
| 历史名校、文化底蕴 | 学院复古 | 棕色、米色 |
| 自然校园、环境介绍 | 清新自然 | 绿色、奶白 |

**根据内容选择布局**：

| 内容特征 | 推荐布局 | 说明 |
|----------|----------|------|
| 有标志性图片要展示 | 分割布局 | 图片占上/左半部分 |
| 信息量大、需要清晰层次 | 卡片镂空 | 图片作背景，内容卡片居中 |
| 图片氛围感强 | 渐变遮罩 | 图片全背景，文字叠加上面 |
| 简洁设计感 | 边框留白 | 内容居中，图片边框装饰 |

### 3.1 背景图片使用规则

**何时使用 bg 图片**：
- 当需要纯色/纹理背景时，使用 `references/bg/` 中的背景图
- 当需要展示大学校园场景时，使用 `references/universities/` 中的大学图片
- 可以同时使用两者：大学图片作主视觉，bg 图片作氛围装饰

**HTML 中引用 bg 图片**：
```html
<!-- 使用 bg 文件夹中的背景图 -->
<div class="background" style="background-image: url('../bg/bg_1.jpg')"></div>

<!-- 或使用大学图片库中的图片 -->
<div class="background" style="background-image: url('../universities/浙江大学/campus-1.jpg')"></div>
```

**背景图片叠加使用示例**：
```html
<!-- 底层：bg 背景图 -->
<div class="bg-layer" style="background-image: url('../bg/bg_5.jpg')"></div>
<!-- 中层：大学图片（可加模糊） -->
<div class="university-layer" style="background-image: url('../universities/浙江大学/campus-1.jpg'); filter: blur(8px)"></div>
<!-- 顶层：内容 -->
<div class="content-layer">...</div>
```

### 4. 规划图片序列

**通用结构**（可根据内容调整）：

| 序号 | 类型 | 作用 |
|------|------|------|
| P1 | 封面 | 主题标题 + 核心亮点 + 主视觉图片 |
| P2-Pn | 内容 | 每张承载1个核心信息点 |
| 最后 | 收尾 | 总结/互动引导（可选） |

**内容分配原则**：
- 每张图只承载 **1个核心信息点**
- 信息量与视觉元素平衡（信息多则图简洁，视觉强则文字精炼）
- 图片与内容匹配（讲宿舍配宿舍图，讲食堂配食堂图）

### 5. 生成 HTML 文件

**为每张卡片生成独立的 HTML 文件，存放在以时间命名的子文件夹中**：

```
output/
└── 20240309153045/      # 生成时间：年月日时分秒（YYYYMMDDHHMMSS）
    ├── P1-cover.html      # 封面
    ├── P2-content.html    # 内容图1
    ├── P3-content.html    # 内容图2
    └── ...
```

**HTML 设计要点**：
- 尺寸：1536 × 2048 px（3:4 @ 2K）
- 字体：中文用系统默认或思源黑体/苹方
- 圆角：8-24px，保持柔和感
- 阴影：轻柔阴影提升层次
- 留白：充足的边距和行距
- 图片：引用从图片库匹配的大学图片
- **自包含**：CSS 内联，图片使用绝对路径或 base64

**HTML 文件特点**：
- 可直接在浏览器打开预览
- 支持打印/截图
- 兼容 Puppeteer/Playwright 等截图工具

### 6. 输出结果

- 在 `output/时间戳/` 文件夹中生成所有 HTML 文件（如 `output/20240309153045/`）
- 提供帖子标题 + 正文 + 话题标签
- 用户可自行将 HTML 转换为 PNG

## HTML 转 PNG 方式

用户可选择以下方式将 HTML 转为图片：

### 方式一：浏览器截图
- 打开 HTML 文件
- 使用截图工具（macOS: Cmd+Shift+4，Windows: Win+Shift+S）

### 方式二：Puppeteer 脚本
```javascript
const puppeteer = require('puppeteer');

async function htmlToPng(htmlPath, outputPath) {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.setViewport({ width: 1536, height: 2048 });
  await page.goto(`file://${htmlPath}`);
  await page.screenshot({ path: outputPath });
  await browser.close();
}
```

### 方式三：在线工具
- html2canvas
- Carbon
- 等

## 内容类型示例

以下是常见内容类型，**仅作参考，不限制用户输入**：

### 大学整体介绍
- 内容：学校概况、排名、特色
- 图片：校门、标志性建筑、全景
- 风格：专业权威或学院复古
- 布局：分割布局或卡片镂空

### 宿舍条件
- 内容：房型、配置、环境
- 图片：宿舍实景、公共区域
- 风格：清新自然或青春活力
- 布局：渐变遮罩或边框留白

### 专业介绍
- 内容：专业特色、课程、就业
- 图片：实验室、课堂、学生活动
- 风格：简约现代或专业权威
- 布局：卡片镂空或分割布局

### 招生数据
- 内容：分数线、计划、政策
- 图片：校园环境、教学楼
- 风格：专业权威
- 布局：渐变遮罩（数据可视化）

### 校园生活
- 内容：社团、活动、日常
- 图片：活动照片、学生生活场景
- 风格：青春活力
- 布局：分割布局或卡片镂空

## API 配置

使用 **NVIDIA API** 调用大模型，优先 **Kimi**。

### 环境变量

```bash
# .env 文件配置
NVIDIA_API_KEY="your-nvidia-api-key"
NVIDIA_BASE_URL="https://integrate.api.nvidia.com/v1"

# 优先模型
KIMI_MODEL="moonshotai/kimi-k2-0711-preview"
```

### 支持的模型

| 模型 | 用途 | 说明 |
|------|------|------|
| Kimi (moonshotai/kimi-k2-0711-preview) | **首选** | 内容分析、HTML 生成 |
| GLM | 备选 | 内容分析、HTML 生成 |
| DeepSeek | 备选 | 内容分析、HTML 生成 |
